# 鼠标自动点击器 — 详细讲解

---

## 一、整体架构

```
┌──────────────────────────────────────────────────┐
│                    main.py                        │
│            (入口：创建窗口、组装模块)                 │
│         ┌──────────┬──────────┬────────┐          │
│         ▼          ▼          ▼        │          │
│    clicker.py  hotkey.py   ui.py      │          │
│   (点击引擎)  (全局热键)  (中文界面)    │          │
│         │          │          │        │          │
│         └──────────┴──────────┴────────┘          │
│                       │                           │
│               pynput (第三方库)                     │
│          mouse.Controller / keyboard.GlobalHotKeys │
└──────────────────────────────────────────────────┘
```

**四个模块各司其职：**
- `main.py` — 只负责创建窗口并启动，代码极少
- `clicker.py` — 纯粹的点击逻辑，跟界面和热键无关
- `hotkey.py` — 纯粹的全局热键管理，跟点击和界面无关
- `ui.py` — 界面组装 + 把上面三者串起来

## 二、各文件逐行讲解

### 1. requirements.txt

```
pynput>=1.7.6
```

为什么只用 pynput 一个库？
- 它同时提供了 **鼠标控制**（`pynput.mouse.Controller`）和 **全局键盘监听**（`pynput.keyboard.GlobalHotKeys`）
- tkinter 是 Python 内置的，无需安装
- 所以整个项目只有这一个第三方依赖

---

### 2. clicker.py — 点击引擎

```
class Clicker:
    _running = threading.Event()   # 线程间通信的"红绿灯"
    _thread  = None                # 点击循环所在的子线程
    _click_count = 0               # 累计点击次数
```

**核心原理：**

```
开始按钮 → Clicker.start()
              → Event.set()          # 把红绿灯设为"绿灯"
              → 创建子线程，运行 _click_loop()

子线程内：
  while Event.is_set():         # 只要绿灯就一直循环
      Controller.click()        # 执行鼠标点击
      time.sleep(interval)      # 等待设定的间隔

停止按钮 → Clicker.stop()
              → Event.clear()        # 红绿灯变"红灯"
              → 子线程检测到红灯，退出循环，线程结束
```

**为什么用 threading.Event 而不是 bool？**
- `Event.is_set()` 是线程安全的，多个线程同时读写不会出 bug
- 普通的 `bool` 在多线程环境下可能因为 CPU 缓存问题导致读取到过期值

**属性设置器（setter）的作用：**
```python
@interval_ms.setter
def interval_ms(self, value):
    self._interval = max(0.01, value / 1000.0)   # 最少 10ms，防止太快
```
- 把毫秒转成秒（内部用秒来 sleep）
- `max(0.01, ...)` 是兜底保护，防止用户输入 0 或负数导致死循环

### 3. hotkey.py — 全局热键

```
class HotkeyManager:
    _listener = None        # pynput 的 GlobalHotKeys 监听器
    _callbacks = {}         # {"<f6>": 回调函数1, "<f7>": 回调函数2}
```

**核心流程：**

```
注册热键:
  register("<f6>", callback)
    → _callbacks["<f6>"] = callback
    → _restart()                       # 重启监听器让新热键生效

_restart():
    → 停掉旧的 listener（如果有的话）
    → 建一个新的 GlobalHotKeys，传入所有已注册的热键
    → listener.start()                 # 启动后台监听

按键时:
  GlobalHotKeys 检测到 F6 按下
    → 调用 callback()
      → ui.py 里的 _hotkey_action()
        → root.after_idle(_toggle)     # 调度到主线程
```

**为什么按键回调要用 `root.after_idle()` 调度？**
- pynput 的键盘回调运行在它自己的子线程里
- tkinter 的 UI 操作（修改 Label 文字、按钮颜色等）必须在主线程执行
- `root.after_idle()` 把函数调用"排队"到主线程的空闲时间执行

**pynput 热键格式：**
| 用户输入 | pynput 格式 |
|---------|------------|
| `F6` | `<f6>` |
| `F7` | `<f7>` |
| `ctrl+f6` | `<ctrl>+<f6>` |
| `shift+alt+f8` | `<shift>+<alt>+<f8>` |

`_normalize_and_bind()` 方法负责把用户输入的格式转成 pynput 能识别的格式。

**界面布局：**

```
┌──────────────────────────────┐
│       鼠标自动点击器 (标题)     │
├──────────────────────────────┤
```

**界面组件选型：**

| 组件 | tkinter 类 | 为什么 |
|------|-----------|--------|

| 下拉框 | `ttk.Combobox` | 限定选项，防止输入错误 |

**关键交互逻辑：**

```python
# 点击"开始/停止"按钮：
_toggle()
  → _read_settings()          # 先把界面上的设置同步到 Clicker
  → clicker.toggle()          # 切换启停
  → _update_ui()              # 更新按钮文字和状态颜色

# 点击计数更新（来自子线程）：
_on_count(count)
  → root.after_idle(lambda: _count_var.set(str(count)))
  #  ^^^^^^^^^^^^^^^ 调度到主线程！因为调用来自 clicker 子线程

# 模式切换：
_on_mode_change()
  → "固定坐标" → _set_fixed_state("normal")   # X/Y输入框可编辑
  → "跟随鼠标" → _set_fixed_state("disabled")  # X/Y输入框变灰
```

### 5. main.py — 入口

```python
root = tk.Tk()
root.title("鼠标自动点击器")
root.resizable(False, False)   # 禁止调整窗口大小
# 居中显示
root.geometry(f"{w}x{h}+{x}+{y}")

clicker = Clicker()
hotkey_mgr = HotkeyManager()
app = App(root, clicker, hotkey_mgr)

root.protocol("WM_DELETE_WINDOW", lambda: (app.shutdown(), root.destroy()))
#  ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^ 点关闭按钮时，先停掉热键和点击线程
```

**启动顺序：**
1. 创建 tkinter 根窗口
2. 创建 Clicker 实例（此时还没开始点击）
3. 创建 HotkeyManager 实例（此时还没开始监听）
4. 创建 App 界面，在 `__init__` 里把热键和界面关联起来
5. `root.mainloop()` 进入事件循环，等待用户操作

## 三、打包为 .exe 详解

### 为什么需要打包？

直接运行 `.py` 文件需要对方安装 Python 和依赖（pynput）。打包成 `.exe` 后，双击就能用，不需要安装任何东西。

### PyInstaller 做了什么？

```
python 源代码       PyInstaller         单个 .exe 文件
main.py     ──┐                    ┌── 鼠标自动点击器.exe
clicker.py  ─┤                    │
hotkey.py   ─┤ 分析 → 打包 → 压缩  ┤   内部结构：
ui.py       ─┤                    │   ├── 启动引导器 (bootloader)
pynput      ─┘                    │   ├── Python 解释器 (嵌入式)
                                  │   ├── 你的 .pyc 字节码
                                  │   ├── pynput 库
                                  │   └── tkinter + Tcl/Tk 运行时
                                  └── 约 12MB
```

### 打包命令解析：

```bash
pyinstaller \
  --onefile \      # 所有东西打包成一个文件（而不是一个文件夹）
  --noconsole \    # 不显示黑色命令行窗口（因为这是 GUI 程序）
  --name "鼠标自动点击器" \  # 输出文件名
  main.py          # 入口脚本
```

### 打包后的文件：

```
dist/
└── 鼠标自动点击器.exe    ← 这个是最终产物，可以复制给任何人使用

build/                   ← 打包过程中的临时文件，可以删除
鼠标自动点击器.spec        ← PyInstaller 的配置文件，可以删除或留着下次用
```

### 为什么 .exe 启动比 .py 慢一点？

因为每次运行时，exe 会把内嵌的 Python 和库文件解压到临时目录，然后再运行。这个解压过程大约需要 1-3 秒。

## 四、技术要点总结

| 要点 | 实现 |
|------|------|
| 不卡界面 | 点击循环跑在子线程 |
| 随时能停 | threading.Event 做线程间信号 |
| 全局热键 | pynput.keyboard.GlobalHotKeys |
| 线程安全 | root.after_idle() 把 UI 更新调度回主线程 |
| 单文件分发 | PyInstaller --onefile |
| 无黑窗口 | PyInstaller --noconsole |
